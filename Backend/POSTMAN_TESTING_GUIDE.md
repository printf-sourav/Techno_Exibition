# Postman Testing Guide

This guide helps you test:

1. Donations + NGO Needs
2. Waste Pickup + Admin Assignment
3. Admin User Approval/Rejection
4. Auto Notifications generated by those actions
5. Hospital role behavior (allowed + forbidden APIs)

## 1. Prerequisites

1. Backend running on `http://localhost:5000`
2. MongoDB connected
3. At least these verified users exist in DB:
  - `retailer` user (donor)
  - `hospital` user
  - `ngo` user (recipient)
  - `admin` user
  - `waste` user (agency)
  - one normal user for approval/rejection test
4. One `InventoryItem` exists for the retailer with quantity > 0

Recommended test credentials:

| Role | Name | Email | Password |
| --- | --- | --- | --- |
| Retailer | Retailer One | retailer@example.com | Password@123 |
| Hospital | City Hospital | hospital@example.com | Password@123 |
| NGO | Hope NGO | ngo@example.com | Password@123 |
| Admin | System Admin | admin@example.com | Admin@123 |
| Requester User | Store Manager | requester@example.com | Password@123 |
| Normal User (pending/approval test) | Test User | user@example.com | Password@123 |

Note: `POST /api/auth/register` currently creates users with role `retailer` by default, so create other roles directly in DB or from your admin/seed flow.

## 2. Create Postman Environment

Create an environment named `MediSync Local` with:

- `baseUrl` = `http://localhost:5000`
- `retailerToken`
- `hospitalToken`
- `ngoToken`
- `adminToken`
- `requesterToken`
- `normalUserToken`
- `ngoUserId`
- `wasteAgencyId`
- `inventoryItemId`
- `needId`
- `donationId`
- `pickupId`
- `pendingUserId`

## 3. Authentication Requests

### 3.1 Login Retailer

- Method: `POST`
- URL: `{{baseUrl}}/api/auth/login`
- Body (JSON):

```json
{
  "email": "retailer@example.com",
  "password": "Password@123"
}
```

In Postman Tests tab:

```javascript
pm.environment.set("retailerToken", pm.response.json().data.token);
```

### 3.2 Login NGO

Use NGO credentials, store as `ngoToken`:

```javascript
pm.environment.set("ngoToken", pm.response.json().data.token);
pm.environment.set("ngoUserId", pm.response.json().data.user._id);
```

### 3.3 Login Admin

Use admin credentials:

```json
{
  "email": "admin@example.com",
  "password": "Admin@123"
}
```

Store as `adminToken`.

### 3.4 Login Requester (pickup creator)

Use requester credentials:

```json
{
  "email": "requester@example.com",
  "password": "Password@123"
}
```

Store as `requesterToken`.

### 3.5 Login Hospital

Use hospital credentials, store as `hospitalToken`:

```json
{
  "email": "hospital@example.com",
  "password": "Password@123"
}
```

```javascript
pm.environment.set("hospitalToken", pm.response.json().data.token);
```

### 3.6 Login Normal User (for admin approval/rejection notification test)

Use normal user credentials:

```json
{
  "email": "user@example.com",
  "password": "Password@123"

```

Store as `normalUserToken`.

## 4. Donations + NGO Need Flow

### 4.1 Create NGO Need

- Method: `POST`
- URL: `{{baseUrl}}/api/donations/needs`
- Header: `Authorization: Bearer {{ngoToken}}`
- Body:

```json
{
  "medicineName": "Paracetamol 650",
  "quantity": 100,
  "urgency": "high",
  "location": "Kolkata"
}
```

Save need id:

```javascript
pm.environment.set("needId", pm.response.json().data._id);
```

### 4.2 Get Needs

- Method: `GET`
- URL: `{{baseUrl}}/api/donations/needs`
- Header: `Authorization: Bearer {{retailerToken}}`

### 4.3 Create Donation (Retailer -> NGO)

- Method: `POST`
- URL: `{{baseUrl}}/api/donations`
- Header: `Authorization: Bearer {{retailerToken}}`
- Body:

```json
{
  "recipientNgoId": "{{ngoUserId}}",
  "inventoryItemId": "{{inventoryItemId}}",
  "ngoNeedId": "{{needId}}",
  "quantity": 25
}
```

Save donation id:

```javascript
pm.environment.set("donationId", pm.response.json().data._id);
```

### 4.4 Verify NGO Notification After Donation

- Method: `GET`
- URL: `{{baseUrl}}/notifications`
- Header: `Authorization: Bearer {{ngoToken}}`

Expected: A notification with:

- `type = "donation_update"`
- `entityType = "donation"`
- `entityId = {{donationId}}`

### 4.5 Mark Donation Distributed

- Method: `PATCH`
- URL: `{{baseUrl}}/api/donations/{{donationId}}/distribute`
- Header: `Authorization: Bearer {{ngoToken}}`

### 4.6 Hospital Role Tests

#### 4.6.1 Hospital can view NGO needs (allowed)

- Method: `GET`
- URL: `{{baseUrl}}/api/donations/needs`
- Header: `Authorization: Bearer {{hospitalToken}}`

Expected: `200 OK`

#### 4.6.2 Hospital can view own donation list (allowed)

- Method: `GET`
- URL: `{{baseUrl}}/api/donations`
- Header: `Authorization: Bearer {{hospitalToken}}`

Expected: `200 OK` with hospital-scoped results (often empty array if no records).

#### 4.6.3 Hospital cannot create donation from retailer inventory (forbidden)

- Method: `POST`
- URL: `{{baseUrl}}/api/donations`
- Header: `Authorization: Bearer {{hospitalToken}}`
- Body:

```json
{
  "recipientNgoId": "{{ngoUserId}}",
  "inventoryItemId": "{{inventoryItemId}}",
  "ngoNeedId": "{{needId}}",
  "quantity": 10
}
```

Expected: `403 Access denied` (route is restricted to `retailer`).

#### 4.6.4 Hospital can request waste pickup (allowed)

- Method: `POST`
- URL: `{{baseUrl}}/api/waste/pickups`
- Header: `Authorization: Bearer {{hospitalToken}}`
- Body:

```json
{
  "wasteType": "expired_medicine",
  "amount": 8,
  "unit": "kg",
  "pickupDate": "2026-04-12T09:00:00.000Z",
  "pickupTime": "09:00 AM",
  "location": "City Hospital Store"
}
```

Expected: `201 Created`

## 5. Waste + Admin Assignment Flow

### 5.1 Create Waste Pickup (Requester)

- Method: `POST`
- URL: `{{baseUrl}}/api/waste/pickups`
- Header: `Authorization: Bearer {{requesterToken}}`
- Body:

```json
{
  "wasteType": "expired_medicine",
  "amount": 15,
  "unit": "kg",
  "pickupDate": "2026-04-10T10:30:00.000Z",
  "pickupTime": "10:30 AM",
  "location": "Howrah Warehouse"
}
```

Save pickup id:

```javascript
pm.environment.set("pickupId", pm.response.json().data._id);
```

### 5.2 Assign Pickup as Admin

- Method: `PATCH`
- URL: `{{baseUrl}}/api/admin/waste/pickups/{{pickupId}}/assign`
- Header: `Authorization: Bearer {{adminToken}}`
- Body:

```json
{
  "agencyId": "{{wasteAgencyId}}"
}
```

### 5.3 Verify Requester Notification After Assignment

- Method: `GET`
- URL: `{{baseUrl}}/notifications`
- Header: `Authorization: Bearer {{requesterToken}}`

Expected: Notification with:

- `type = "pickup_assigned"`
- `entityType = "pickup"`
- `entityId = {{pickupId}}`

## 6. Admin Approval/Rejection Notification Flow

### 6.1 Get Pending Users

- Method: `GET`
- URL: `{{baseUrl}}/api/admin/users/pending`
- Header: `Authorization: Bearer {{adminToken}}`

Pick a user id and set `pendingUserId`.

### 6.2 Approve User

- Method: `PATCH`
- URL: `{{baseUrl}}/api/admin/users/{{pendingUserId}}/approve`
- Header: `Authorization: Bearer {{adminToken}}`

Alternative reject endpoint:

- `PATCH {{baseUrl}}/api/admin/users/{{pendingUserId}}/reject`

### 6.3 Verify User Notification

Login as that user and call:

- Method: `GET`
- URL: `{{baseUrl}}/notifications`
- Header: `Authorization: Bearer {{normalUserToken}}`

Expected:

- On approve: `type = "verification_update"` with approval message
- On reject: `type = "verification_update"` with rejection message

## 7. Optional: Mark Notification Read

- Method: `PUT`
- URL: `{{baseUrl}}/notifications/<notificationId>`

## 8. Common Failure Cases

1. `401 Invalid token`: missing/expired bearer token
2. `403 Access denied`: token role does not match endpoint requirement
3. `404 Inventory item not found for donor`: wrong `inventoryItemId` or item not owned by retailer
4. `400 Insufficient inventory quantity for donation`: donation quantity exceeds available stock
5. `404 Waste agency user not found`: wrong `wasteAgencyId` or user role is not `waste`

## 9. Suggested Collection Order

1. Login all roles
2. Create NGO need
3. Create donation
4. Verify NGO notifications
5. Run hospital role tests
6. Create pickup
7. Assign pickup (admin)
8. Verify requester notifications
9. Approve/reject user (admin)
10. Verify user notifications
